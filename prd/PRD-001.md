# Product Requirements Document (PRD): Unsloth MoE Fine-Tuning Pipeline

## 1. Objective
Create a localized environment to prepare a training dataset, write unit tests for the data pipeline, and construct a self-contained Docker image that uses the Unsloth framework to fine-tune a Mixture-of-Experts (MoE) LLM (defaulting to Mixtral 8x7B) using QDoRA/QLoRA.

## 2. Tech Stack
- **Languages:** Python 3.10+
- **Package Manager:** `uv`
- **Testing:** `pytest`
- **ML Frameworks:** `unsloth`, `transformers`, `trl`, `peft`
- **Infrastructure:** Docker

## 3. Tasks & Deliverables

### Task 1: Dataset Preparation & Testing
- Create a sample dataset file `data/sample_dataset.jsonl` containing at least 5-10 dummy conversational examples (e.g., system/user/assistant formatting).
- Write a Python script `src/prepare_data.py` that loads this JSONL file and formats it into a standard Hugging Face `Dataset` object (apply ChatML or Alpaca formatting).
- Write a test suite `tests/test_data.py` using `pytest` to verify that:
  - The JSONL file loads correctly.
  - The data formatting function correctly maps the input fields to the expected output prompt format.
  - No empty or malformed strings are returned.

### Task 2: Unsloth Training Script
- Create a Python script `src/train.py` that utilizes Unsloth to run a brief fine-tuning job (1 epoch or a few steps for testing).
- **Requirements for the script:**
  - Use `argparse` or environment variables to accept a `model_id`. Default this to `mistralai/Mixtral-8x7B-v0.1`.
  - Initialize the model using Unsloth's `FastLanguageModel.from_pretrained`. Ensure 4-bit quantization (`load_in_4bit=True`) is enabled.
  - Apply QLoRA/QDoRA adapters via `FastLanguageModel.get_peft_model` (enable `use_dora=True`).
  - Load the embedded dataset generated from Task 1.
  - Use `SFTTrainer` from the `trl` library to execute the training run.
  - Save the resulting LoRA adapters to an output directory `/workspace/output`.

### Task 3: Docker Containerization
- Write a `Dockerfile` that packages the entire workflow.
- **Dockerfile constraints:**
  - Base image: Must be a CUDA-compatible image capable of running Unsloth (e.g., `pytorch/pytorch:2.2.0-cuda12.1-cudnn8-devel` or an official Unsloth base image).
  - Copy the `data/`, `src/`, and `tests/` directories into the container.
  - Install dependencies natively using `uv pip install ...`.
  - Embed the dataset directly into the image for now (no external volume mounts required for the data).
  - Set the default `ENTRYPOINT` to execute `python src/train.py`. Allow overriding the model ID via command arguments.

## 4. Acceptance Criteria
1. Running `uv run pytest tests/` passes successfully on the local machine.
2. The Docker image builds successfully without dependency conflicts.
3. Running the Docker container executes the `train.py` script and initializes the Unsloth model pipeline. *(Note: Execution will crash if run on macOS due to missing NVIDIA GPUs, but the agent must ensure the build and code logic are perfectly valid for an x86_64 CUDA environment).*
