# Product Requirements Document (PRD): Axolotl MoE Fine-Tuning Pipeline

## 1. Objective

Create a localized environment to prepare a training dataset, write unit tests for the data pipeline, and construct a self-contained Docker image that uses the Axolotl framework to fine-tune a Mixture-of-Experts (MoE) LLM (defaulting to a 4-bit Mixtral 8x7B) using QDoRA/QLoRA.

## 2. Tech Stack

* **Languages:** Python 3.10+
* **Package Manager:** `uv`
* **Testing:** `pytest`
* **ML Frameworks:** `axolotl`, `transformers`, `peft`, `bitsandbytes`
* **Infrastructure:** Docker

## 3. Tasks & Deliverables

### Task 1: Dataset Preparation & Testing

* Create a sample dataset file `data/sample_dataset.jsonl` containing at least 5-10 dummy conversational examples.
* Write a Python script `src/prepare_data.py` that loads this JSONL file and formats it into an Axolotl-compatible JSONL format (e.g., standard Alpaca or ShareGPT format) to be consumed by the YAML config.
* Write a test suite `tests/test_data.py` using `pytest` to verify that:
* The JSONL file loads correctly.
* The data formatting function correctly maps the input fields to the expected output schema (e.g., containing `instruction`, `input`, and `output` keys, or `conversations` arrays).
* No empty or malformed strings are returned.



### Task 2: Axolotl Training Configuration

* Create a YAML configuration file `config/train.yml` that instructs Axolotl to run a brief fine-tuning job (1 epoch or a few steps for testing).
* **Requirements for the YAML script:**
* Set the `base_model` to `unsloth/mixtral-8x7b-v0.1-bnb-4bit` (to bypass CPU RAM spikes by using a pre-quantized model).
* Enable 4-bit loading (`load_in_4bit: true`).
* Configure the adapter for QDoRA (`adapter: qlora` and `peft_use_dora: true`).
* Point the `datasets` section to the formatted JSONL file generated from Task 1.
* Set the `output_dir` to `/workspace/output`.
* Include basic training hyperparameters (e.g., `micro_batch_size: 1`, `gradient_accumulation_steps: 4`, `gradient_checkpointing: true`).



### Task 3: Docker Containerization

* Write a `Dockerfile` that packages the entire workflow.
* **Dockerfile constraints:**
* Base image: Leverage an official Axolotl CUDA-compatible image (e.g., `winglian/axolotl:main-py3.10-cu121-2.1.2` or similar) to ensure all customized flash-attention and deepspeed dependencies are pre-compiled.
* Copy the `data/`, `src/`, `tests/`, and `config/` directories into the container.
* Install any additional testing/data prep dependencies natively using `uv pip install ...`.
* Embed the dataset directly into the image for now (no external volume mounts required for the data).
* Set the default `ENTRYPOINT` to execute Axolotl's CLI with the config file: `accelerate launch -m axolotl.cli.train config/train.yml`.

## 4. Acceptance Criteria

1. Running `uv run pytest tests/` passes successfully on the local machine.
2. The Docker image builds successfully without dependency conflicts.
3. Running the Docker container successfully invokes the Axolotl CLI, parses the YAML file, and begins initializing the training pipeline. *(Note: Execution will crash if run on macOS due to missing NVIDIA GPUs, but the agent must ensure the build and configuration logic are perfectly valid for an x86_64 CUDA environment).*
